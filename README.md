### Database Schema

The database includes the following tables:

* `pokemon_entity`: Stores Pokémon information, including name, description, types, abilities, height, and weight.
* `pokemon_types`: Stores the types associated with each Pokémon.
* `pokemon_abilities`: Stores the abilities of each Pokémon.

You can visualize the database schema using tools like  **dbdiagram.io** .

### Final Notes

This project is a basic implementation of a Pokedex API with **CRUD** functionality for managing Pokémon favorites. It uses **Spring Boot** for the backend, **PostgreSQL** for the database, and **React** for the frontend (optional).
